> 이 파일 내용을 통째로 복사해서 안티그래비티 새 세션 첫 메시지로 붙여넣으세요.

---

## [프로젝트 컨텍스트]

A-Ka 프로젝트는 FastAPI + Celery + LangGraph + pgvector 기반의 한국어 카카오톡 챗봇 백엔드입니다.
사용자가 유튜브 링크를 보내면 자막을 추출→청킹→요약→임베딩해서 DB에 저장하고,
RAG로 검색 가능한 지식 베이스를 만드는 시스템입니다.

- 루트 경로: `C:\A-ka_backend`
- 말투: 반드시 **존댓말** 유지
- 역할 분리: 채훈(나) = 파이프라인/어댑터, 유리님 = 모델/Repository/마이그레이션

---

## [기술 스택]

- FastAPI + Celery (chain/shared_task) + Redis
- LangGraph StateGraph (IntelligenceState TypedDict)
- OpenAI Structured Output (VideoOverview — 11개 한국어 카테고리 + free fallback)
- Async SQLAlchemy 2.0 (`async_session_maker`, AsyncSession)
- pgvector Vector(1536) — **아직 모델에 미반영 (블로커)**
- Repository 패턴 + pipeline/repo 경계의 key adapter

## [대화 톤 / 작업 스타일 주의사항]

- 반드시 **존댓말 사용** (반말 금지)
- 파일 수정 시 **한국어 주석 유지**
- "채훈", "유리님" 역할 분리 항상 명시
- 카테고리는 **A안**(Repository 내부 upsert) — 이미 확정, 뒤집지 말 것
- 테이블명은 **ERD 기준** — `knowledge_chunk` (YoutubeKnowledgeChunk 클래스 내부 `__tablename__`)
- 코드 작성 시 `async with async_session_maker()` 패턴 유지 (세션 누수 방지)
- Celery sync ↔ async DB는 **반드시 `run_async` + `_repo_*` 래퍼** 경유
